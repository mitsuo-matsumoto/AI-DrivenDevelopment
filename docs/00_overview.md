# 00_overview.md

# 1. プロジェクト概要

本プロジェクトは、個人用のタスク管理Webアプリを作成することを目的とする。

既にBacklogやWrikeなどのタスク管理ツールは利用しているが、
それらとは別に「手元メモとして素早く記録・整理できる軽量なツール」を想定している。

業務管理ではなく、個人の思考整理・即時記録用途を主目的とする。

---

# 2. 対象ユーザー

- 本人のみ（個人利用）
- ログイン機能は持たない
- データ同期は行わない（単一ブラウザ利用）

---

# 3. 目的

本アプリの主な目的は以下とする。

- タスクの登録・編集・削除
- タスクの完了管理
- カテゴリやタグによる整理
- シンプルで直感的なUI
- サーバ不要でローカル完結

---

# 4. 技術スタック

## 4.1 フロントエンド

- React（SPA構成）

## 4.2 状態管理

React Hooksを利用する。

### ■ 状態管理の考え方

Reactにおける状態管理とは、
「アプリ内のデータ（状態）が変わったときにUIも自動的に更新される仕組み」である。

本プロジェクトでは用途に応じて以下を使い分ける。

- useState  
  単純なUI状態を管理  
  例：
  - 入力フォームの値
  - モーダルの開閉状態
  - フィルターの選択状態

- useReducer  
  タスクリスト全体の状態を管理  
  例：
  - タスク追加
  - タスク削除
  - ステータス変更
  - 一括更新処理

アプリの中核となるデータはuseReducerで一元管理する。

---

## 4.3 データ永続化

- ブラウザのLocalStorageを利用する
- サーバは使用しない
- 将来的にAPIサーバ連携へ拡張可能な構造とする

---

# 6. UI / UX 方針

- 付箋のような軽い操作感
- タスクの状態を視覚的に区別する
  - 未着手
  - 処理中
  - 完了
- 完了タスクは通常非表示とする
- 完了から1日経過したタスクは自動削除される
- カテゴリによるフィルタリング機能を持つ

「素早く書けて、素早く消える」軽量さを重視する。

# 7. 用語定義

| 用語 | 説明 |
|------|------|
| タスク | ユーザーが登録する作業項目 |
| カテゴリ | タスクを分類するラベル |

| 完了 | タスクが終了した状態 |
| LocalStorage | ブラウザ内にデータを保存する仕組み |
| Reactコンポーネント | UIを構成する再利用可能な部品 |
